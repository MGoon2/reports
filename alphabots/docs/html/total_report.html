5. 핵심 위험

등급위험대상요약
Critical예외 시 실행 턴 이력 불완전CollectData, Finance, RadarSTART 후 catch 경로에서 STOP/ERROR 이력이 누락될 수 있다.
Critical비밀값/접속정보 config 집중전체, 특히 StockDB, Token, Key, Pwd 계열 키가 설정 파일에 존재한다. 값은 본 보고서에 없다.
CriticalStock 초기화 실패 전파StockProcess 생성자에서 AWS/Firebase/DB성 작업이 즉시 초기화된다.
High빈 catch/침묵 실패전체장애 원인, OIdx/OTIdx, 작업명 로그가 누락되는 경로가 많다.
HighThread 직접 생성 및 공유 상태전체actionState 직접 공유로 race/중복 실행 가능성이 있다.

6. 리팩터링 우선순위

  1. 실행 이력 표준화: 모든 프로젝트를 START → ERROR? → END/STOP finally 패턴으로 맞춘다.
  2. 빈 catch 제거: 최소 OIdx/OTIdx/작업명/예외 타입/메시지를 로그에 남긴다.
  3. Secret 외부화: config 값은 vault/환경변수 참조로 이전한다.
  4. 스케줄러 추출: DispatcherTimer/UI와 worker 실행을 분리한다.
  5. SP catalog 작성: Proc/USP 호출을 프로젝트·작업·입출력 기준으로 관리한다.
  6. 외부 API 표준화: timeout, retry, backoff, circuit breaker를 적용한다.
