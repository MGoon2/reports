ALB TG health check port + prometheus scrape target도 8186로 조정. scrape payload 경량화 (메트릭 description 생략) — broken pipe 재발 보조 방지.

4. Dockerfile 설정

FROM amazoncorretto:25

ENV TZ=Asia/Seoul
RUN ln -sf /usr/share/zoneinfo/Asia/Seoul /etc/localtime

WORKDIR /app

ARG JAR_FILE=build/libs/*.jar
COPY ${JAR_FILE} app.jar

EXPOSE 8086 8186

ENTRYPOINT ["sh","-c","java -Duser.timezone=Asia/Seoul ${JAVA_OPTS:-} -jar /app/app.jar"]

Dockerfile 설명

  • Base Image: amazoncorretto:25 (Amazon Corretto JDK 25)
  • Timezone: Asia/Seoul
  • Work Directory: /app
  • JAR File: build/libs/*.jar
  • Exposed Ports: 8086 (Application), 8186 (Actuator)
  • Entrypoint: java -Duser.timezone=Asia/Seoul ${JAVA_OPTS:-} -jar /app/app.jar

  • DataSource 연결 정보 확인 (MSSQL, PostgreSQL, MongoDB)
  • S3 설정 확인 (FINUP_S3_ENABLED, FINUP_S3_BUCKET, FINUP_S3_BASE_URL)

  • ALB TG health check port를 8186로 조정

트러블슈팅

DB 연결 실패

  • DataSource URL 확인 (host, port, database name)
  • 자격 증명 확인 (username, password)
  • HikariCP pool 설정 확인 (max-size: 30, min-idle: 5)
  • leak detection 로그 확인 (60초)

S3 연결 실패

  • FINUP_S3_ENABLED 확인
  • FINUP_S3_BUCKET 확인
  • FINUP_S3_REGION 확인 (ap-northeast-2)
  • AWS 자격 증명 확인 (profile 또는 IAM role)

Actuator 접근 실패
