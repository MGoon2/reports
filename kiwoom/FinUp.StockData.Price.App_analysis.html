FinUp.StockData.Price.App Analysis Report

Date: 2026-05-07 · Baseline setting: StartUp=KR · Target: C:\Dev\FinUp.StockData.Price.App

Executive Summary

Kafka

No evidence of active Kafka use in the KR runtime path. Kafka settings remain, but UseLog=false prevents the Core.Log Kafka DAL from being created.

Redis / Valkey

Redis Pub/Sub code exists, but KR/PRICE auto distribution is currently disabled. The configured endpoint already points at a Valkey-looking AWS endpoint.

MongoDB / PostgreSQL

Mongo writes are disabled for the current KR/PRICE configuration. The Mongo PRICE handler start call is also commented out, so enabling only the flag would not start persistence.

With the current StartUp=KR, FIDType=PRICE, RedisDistribute=false, and MongoSaveData=false settings, the app receives Kiwoom real-time price events but does not automatically publish them to Kafka/Redis or persist them to MongoDB.

KR Runtime Flow

  1. Program.Main reads StartUp and runs SchedulerKiwoomReal for KR.
  2. SchedulerKiwoomReal reads MarketType, FIDType, and schedule settings, then creates KiwoomRedis.
  3. KiwoomRedis creates KiwoomClient and receives real-time events through OnPrice, OnHoga, and OnTrader.
  4. Because FIDType=PRICE, the current path focuses on price events. Redis and Mongo handlers are not created under the current flags.

Kafka Findings

  • KafkaUrl and LogCommunication=kafka remain in config, but UseLog=false. App.config:101-107
  • The target app has no direct FinUp.Core.Log or Logger.Instance usage found by repository search.
  • FinUp.Core.Data can call Logger.Instance, but Logger only creates a logging DAL when UseLog=true. Sql.cs:84-95, Log.cs:16-62, Logger.cs:22-30
  • FinUp.Core.Log still references Confluent.Kafka, so Kafka remains a build/deployment dependency even if runtime logging is disabled. FinUp.Core.Log.csproj:89-91

Inference: Kafka is effectively unused in the KR runtime. Removing Kafka operationally is low risk, but removing the package/reference requires a separate dependency cleanup decision.

Redis / Valkey Findings

  • The app defines stockdata and stockdatapubsub clusters under a custom finup config section. App.config:4-33
  • The active publishing handler uses RedisDatabase.GetSubcriber("stockdatapubsub") and publishes to PRICE, HOGA, and TRADER channels. RedisStockDataHandler.cs:24-51, 73-121
  • For the current PRICE path, this handler is only created when RedisDistribute=true. It is currently false. App.config:98, KiwoomRedis.cs:71-78
  • If FIDType is changed away from PRICE, the HOGATRADER branch creates Redis regardless of RedisDistribute. KiwoomRedis.cs:87-98, 133-140
  • Core.Redis uses StackExchange.Redis 2.1.58 and does not expose SSL/AUTH settings in its config schema. FinUp.Core.Redis.csproj:75-80, RedisCluster.cs:138-149

Inference: AWS Valkey compatibility is likely for plain Redis protocol Pub/Sub, but TLS/auth-enabled Valkey would require config/code support that is not present here.

MongoDB / PostgreSQL Findings

  • Mongo settings remain in config, but MongoSaveData=false. App.config:78-81, 99
  • The PRICE Mongo handler is only created if MongoSaveData=true. KiwoomRedis.cs:80-85
  • The PRICE Mongo handler start call is commented out. Enabling the flag alone would create a handler but not start its consumer. KiwoomRedis.cs:126-131
  • MongoStockDataHandler writes to year-suffixed databases and collections such as T_Price, T_Hoga, and T_Trader. MongoStockDataHandler.cs:27-43, 86-117
  • Migration/test/alternate startup paths still contain direct Mongo usage, so a full Mongo-to-PostgreSQL conversion is broader than the KR runtime path.

Inference: For KR/PRICE only, MongoDB is currently not an active runtime dependency. For full app parity, PostgreSQL migration needs a wider inventory of migration forms, test forms, and alternate scheduler modes.

Unknowns / Limits

  • Actual AWS Valkey network reachability, TLS, and auth settings were not proven by repository analysis.
  • Whether Kafka DLLs are copied into deployment output requires a build/deployment artifact check.
  • The PostgreSQL migration scope needs a decision: KR real-time price only, or every Mongo-using mode and migration tool.
  • App.config defines KiwoomStartDelay twice; this should be checked separately because duplicate appSettings keys can create runtime configuration risk. App.config:91, 100
