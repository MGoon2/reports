이 워크스페이스의 여러 C#/.NET 프로젝트를 분석해라.
분석할 내용은 `FinUp.*AlphaBot*` 들이고 프로젝트 명에 Test가 들어가 있다면 제외 시켜도 된다.

중요:
- 코드는 절대 수정하지 마라.
- 빌드, 실행, 패키지 설치는 사용자의 허락 없이 하지 마라.
- 운영 DB/API/외부 시스템에 접근하지 마라.
- 비밀번호, connection string, API Key 값은 출력하지 마라.

분석 목표:
1. 전체 프로젝트 목록과 역할 파악
2. 프로젝트 간 참조 관계 파악
3. 실행 프로젝트와 라이브러리 프로젝트 분류
4. 각 프로젝트의 진입점 파악
5. 반복 루프, 타이머, 스케줄러, BackgroundWorker, Task, Thread 사용 위치 파악
6. DB 접근 코드 파악
7. Stored Procedure 호출 위치 파악
8. 설정 파일 사용 방식 파악
9. 로그 처리 방식 파악
10. 예외 처리 방식 파악 
11. 예외가 발생 할 경우 턴이 완전 종료되는 경우는 크리티컬로 분류
13. `total_report.html`은 앞서 작성된 각 md 파일들의 종합 분석 보고서다. 보고서는 $codex-design binance로 한다.

각 문서는 표와 Mermaid 다이어그램을 적극 활용해라.

분석 순서:
1. 먼저 전체 파일/프로젝트 구조를 읽어라.
2. .sln, .csproj 파일을 기준으로 프로젝트 관계를 파악해라.
3. Program.cs, Main, App.xaml, Form 초기화 코드 등 진입점을 찾아라.
4. DB 접근 코드를 검색해라.
5. 루프/타이머/스레드 관련 코드를 검색해라.
6. 위험 요소를 정리해라.
7. 마지막에 리팩터링 계획을 작성해라.
