0. 결론 (TL;DR)

  1. 외부 요청 플로우는 사용자 정보로 확정됨: Route53 → ELBMainExternal → FINUP WAF EC2(2대) → ELBMainInternal → 레거시 CS EC2 또는 신규 TG. ELBMainExternal/MarsExternal은 거의 패스스루이고 실제 URL 분기는 ELBMainInternal에 집중 (포트 80/8443에 99 rules + 2 default).
  2. 이미 신규 TG로 라우팅되는 URL이 13개: www/finup/biz/finance/mfinance/m/mstock/mobileappstock/stock/stockdata/mentorstock/company/freechat.finup.co.kr — 레거시 ALB는 통과만 하고 백엔드는 신규 React/Java. Tier 2의 1순위 후보.

2.2 ELBMainInternal :8443 — 39 rule (대부분 prod API용)

거의 INT_80과 미러. 차이점: - :80에 있는 신규 React TG-front--prod 9개 rule이 :8443에 없음 (즉 신규 prod React는 :80만 받음) - :8443에는 prod API + pre-가 주로 매핑 (구 클라이언트가 8443으로 호출) - apilogtest.finup.co.kr rule 28만 :8443에 존재 (TG ELBMainInternalTargetApiLogTest, API Log2 공유) - *.financeup.co.kr pre.financeup.co.kr rule 6 → StagingWEBAPI (unhealthy) - preapi.stockpoint.co.kr rule 36 → StagingWEBAPI (unhealthy)

외부 → WAF EC2 → ELBMainInternal:80 또는 :8443 어느 포트로 forwarding되는지 패턴은 WAF 설정에 달림. WAF 측에서 apistock/apiauth 등은 :8443으로, 일반 웹은 :80으로 분기시키는 것으로 추정 (별도 확인 필요).

2.3 ELBMainExternal :80/:443 (passthrough)

  • :80 — 30 rule이 모두 HTTPS 301 redirect. 단순 80→443 정책.
  • :443 — 24 rule (대부분 도메인 통합 redirect: financeup.co.kr → www.financeup.co.kr, stockpoint.co.kr → stock.finup.co.kr 등). 실제 forward는 4건만:
  • grafana.finup.co.kr → TG-monitoring-grafana (rule 16, 19, 21, 22, 23 5중 등록)
  • apistock.finup.co.kr /api/chat/weblogin* → TG-api-stock-external (rule 18)
  • apistock.finup.co.kr /api/chat/signal /api/chat/signal/last* → TG-api-stock-external (rule 100, 101)
  • apistock/mstock/mobileappstock /Stock/Banner/* → redirect img.finup.co.kr (rule 14)
  • 그 외 모두 default → ELBMainExternalTarget → WAF EC2

TG-api-stock-external은 신규 Java api-stock 서비스로 보임. 일부 chat 관련 endpoint(/api/chat/weblogin, /api/chat/signal)가 외부 ALB에서 직접 라우팅됨. Bypassing WAF — 의도적인지 확인 필요.
