XML 조건식 → 종목 평가 본체 동작 방식

XML_TEST/ 폴더의 4개 샘플 파일(60일선4_20240611_2.xml, 보컬눌림목4_20240611.xml, 역매공파4_20240604.xml, 오돌이4_20240815_순서변경.xml)을 기준으로, XML 한 줄이 어떻게 메모리 객체로 변환되어 종목 데이터(Constants.StockDay)에 적용되는지 단계별로 추적한 문서.

0. TL;DR – 한 종목, 한 날짜에 일어나는 일

XML 파일
   │ ① XmlSerializer
   ▼
ExpressionRoot
  └─ Expressions: List<ExpressionMapper>     ← 루트 식들 (AND 조합)
        └─ Child: List<ExpressionMapper>     ← 그룹식의 자식들
   │ ② DynamicExpression.BindExpression  → TreeView
   │ ③ ExpressionExcutableBinder(재귀)
   │       · Key 문자열 → 어셈블리 리플렉션으로 IExpression 구현체 매칭
   │       · Activator.CreateInstance + SetParam(<UIValue>) + Init()
   ▼
List<ExpressionExcutebleWrap>                ← 실행 가능 트리
   │ ④ for stock in Constants.StockDay
   │     for date in [start..end]
   │       ExpressionLoopCore(stock, date)
   ▼
   ┌──────────────────────────────────────────────────────────────┐
   │  루트 식들을 순서대로 평가 (전부 true → 종목 추출)           │
   │                                                              │
   │  · IsGroup=false              → ICondition.Condition         │
   │  · IConditionGroupOr           → 자식 중 하나만 true면 OK     │
   │  · IGroupRangeIndex            → 범위 인덱스 그룹             │
   │  · IGroupLine                  → 라인 그룹                    │
   │  · 일반 그룹(IGetRange)        → 범위(Tick 리스트) 산출 →    │
   │     자식 ISetRange에 주입 → 슬라이딩 윈도우로                │
   │     IConditionGroup.ConditionGroupRange + 자식 ICondition    │
   │                                                              │
   │  · ISuccess(SUC/SIM_SUC/...)   → 위 평가 후 보유기간 수익률   │
   │     백테스트로 성공/실패/구간 수익률 계산                     │
   └──────────────────────────────────────────────────────────────┘

평가 본체 핵심 함수: - 트리 변환: Analysis/DynamicExpression.cs:182-235 - 종목 루프: Analysis/DynamicExpression.cs:342-385 - 조건 분기: Analysis/DynamicExpression.cs:453-499 - 비교 연산자: Common/Util.cs:790-814 - v3 캔들 비교: Analysis/Expression/v3/Common/Util.cs:100-220 - v3 값 추출(GetTickValue): Analysis/Expression/v3/Common/Extensions.cs:267-392

1. XML 스키마 표준형

XML_TEST/의 모든 파일은 동일한 구조를 따른다.

<ExpressionRoot>
  <ExpressionName>...</ExpressionName>
  <ExpressionVersion>1</ExpressionVersion>
  <Expressions>
    <ExpressionMapper>           <!-- 루트 식: 항상 AND로 묶임 -->
      <Key>PRC_COMP</Key>        <!-- 식 종류 (필수) -->
      <Title>가격단순</Title>
      <IsAndOperator>true</IsAndOperator>
      <IsCondition>true</IsCondition>
      <IsGroup>false</IsGroup>
      <HasOutValues>false</HasOutValues>
      <Param>
        <UIValue><Key>TickType</Key><Value>DAY</Value></UIValue>
        <UIValue><Key>LeftTickType1</Key><Value>20MA</Value></UIValue>
        <UIValue><Key>LeftRate1</Key><Value>1</Value></UIValue>
        <UIValue><Key>Operator1</Key><Value>gt</Value></UIValue>
        <UIValue><Key>RightTickType1</Key><Value>120MA</Value></UIValue>
        <UIValue><Key>RightRate1</Key><Value>1</Value></UIValue>
        ...
      </Param>
      <Child>
        <!-- 그룹식인 경우 자식 ExpressionMapper들이 여기에 -->
      </Child>
    </ExpressionMapper>
    ...
  </Expressions>
</ExpressionRoot>

핵심 포인트: - Key 가 곧 식별자. C# 측 IExpression 구현체의 public string Key => "PRC_COMP" 와 매칭된다. - Param 은 UI 컨트롤별로 정의된 UIValue(Key, Value) 리스트. 의미는 컨트롤마다 다르다. - Child 는 그룹식(IsGroup=true)에만 채워진다. 자식 자체도 ExpressionMapper이므로 임의 깊이 중첩 가능. - IsAndOperator 는 사실상 모두 true. 루트 레벨은 항상 AND, OR이 필요하면 GRP_OR 그룹식으로 묶는다.

3. XML → 메모리 객체 변환 (1~3단계)

3.1 ① XML 역직렬화

DynamicExpression.BindExpression(path) 내부에서 XmlSerializer<ExpressionRoot>로 파일을 읽어 ExpressionRoot.Expressions: List<ExpressionMapper> 를 얻는다.

// 의사코드
using var fs = File.OpenRead(path);
var ser = new XmlSerializer(typeof(ExpressionRoot));
ExpressionRoot root = (ExpressionRoot)ser.Deserialize(fs);

ExpressionMapper 정의는 Common/ExpressionNode.cs:66-136. Param은 List<UIValue>(키-값 쌍)로 보존된다. Child는 자기 자신 타입의 리스트라 트리 구조 그대로 매핑.

XML의 오타 보정 한 줄: Key.StartsWith("RPC") 면 자동으로 PRC_*로 치환 (ExpressionNode.cs:122-124).

3.2 ② TreeView 바인딩

ExpressionMapper.GetNode()가 ExpressionNode(WinForms TreeNode 파생)를 반환하고, 재귀로 자식까지 모두 트리에 추가. 사용자가 보는 좌측 트리 = XML 트리.

3.3 ③ 실행 트리 빌드 — ExpressionExcutebleWrap 트리

이 단계가 XML과 실제 평가 코드를 잇는 핵심.

// Analysis/DynamicExpression.cs:182-235
private IExpression CloneExpression(ExpressionMapper map)
{
    var type = _expressions.First(a => a.Key == map.Key).GetType(); // ← Key로 타입 룩업
    var exp  = (IExpression)CreateControl(type);                    // Activator.CreateInstance
    exp.SetParam(map.Param);                                        // <UIValue> 리스트 → UI 컨트롤 값 세팅
    exp.Init();                                                     // 컨트롤 값 → private 필드 캐싱
    return exp;
}

public ExpressionExcutebleWrap ExpressionExcutableBinder(ExpressionNode node)
{
    var exp = new ExpressionExcutebleWrap { Expression = CloneExpression(node.Mapper) };
    foreach (ExpressionNode child in node.Nodes)
        exp.ChildExpression.Add(ExpressionExcutableBinder(child));   // 재귀
    return exp;
}

_expressions는 어셈블리 리플렉션으로 한 번만 수집한 모든 IExpression 구현체 인스턴스 목록 (DynamicExpression.cs:64-99).

_expressions = (from t in assembly.GetTypes()
                where typeof(IExpression).IsAssignableFrom(t) && t.IsClass
                select Activator.CreateInstance(t) as IExpression).ToList();

→ 즉 Key="PRC_COMP"라는 문자열 한 개가 곧 ucPrice 클래스 인스턴스로 부활한다. (Analysis/Expression/v3/Price/ucPrice.cs:24)

XML의 <UIValue><Key>LeftTickType1</Key><Value>20MA</Value></UIValue> 가 컨트롤 내부 private string _leftType1 = "20MA" 로 자리잡는 과정.

4. 평가 본체 — ExpressionLoopCore

Analysis/DynamicExpression.cs:453-499

private bool ExpressionLoopCore(RoboStock stock, DateTime date)
{
    bool isTrue = false;
    foreach (var exp in _expressionExcutable.Where(a => !(a.Expression is ISuccess)))
    {
        if (!exp.Expression.IsGroup)
            isTrue = ((ICondition)exp.Expression).Condition(stock, date);                 // (A)
        else if (exp.Expression is IConditionGroupOr)
            isTrue = exp.ChildExpression.Any(c => ((ICondition)c.Expression).Condition(stock, date));  // (B)
        else if (exp.Expression is IGroupRangeIndex)
            isTrue = GetConditionGroupRangeIndex(stock, date, exp);                       // (C)
        else if (exp.Expression is IGroupLine)
            isTrue = GetConditionGroupLine(stock, date, null, exp);                       // (D)
        else
        {
            var range = ((IGetRange)exp.Expression).GetRange(stock, date);                // (E)
            foreach (var s in exp.ChildExpression.OfType<ISetRange>())
                s.SetRange(stock, range, date);

            foreach (var tick in range)
            {
                var rangeFowardMoving = range.Where(a => a.date >= tick.date).ToList();
                isTrue = ((IConditionGroup)exp.Expression)
                            .ConditionGroupRange(stock, rangeFowardMoving, date, range.Count);
                if (isTrue)
                    isTrue = exp.ChildExpression.All(c =>
                        ((ICondition)c.Expression).Condition(stock, tick.date));
                if (isTrue) break;
            }
        }
        if (!isTrue) break;   // ← 루트는 AND. 하나라도 false면 즉시 종목 탈락
    }
    return isTrue;
}

5가지 분기 의미를 4개 샘플 XML 식별로 매핑하면 다음과 같다.

분기 인터페이스 예시 XML 식
(A) 단일 ICondition PRC_COMP, PRC_COMP_RANGE, PRC_COMP_RANGE_MINMAX, VAL_COMP_CONST, VAL_COMP_RANGE_AVG, STS_STS, OTH_60LINE_COMP, IND_BOL, IND_BOL_RANGE, BAND_LINE_PRICE_COMP, SIM_GAP, CDL_COMP_CONST
(B) OR IConditionGroupOr GRP_OR (역매공파)
(C) 인덱스 그룹 IGroupRangeIndex (현재 v3에서 OTH_60LINE_RANGE는 일반 그룹)
(D) 라인 그룹 IGroupLine GRP_LINE (역매공파, 오돌이)
(E) 일반 그룹 IConditionGroup+IGetRange OTH_60LINE_RANGE (60일선)

루트 레벨은 항상 AND — 한 식이라도 false면 종목 탈락 (line 499 if(!isTrue) break;).

private static bool CheckCondition(RoboStock stock, DateTime date, int leftTickCount, string leftType, double leftRate,
                                   int rightTickCount, string rightType, double rightRate, string op)
{
    if (string.IsNullOrEmpty(leftType) || string.IsNullOrEmpty(rightType)) return true; // 미사용 슬롯은 통과

    var leftTick  = stock.ticks.Tick(date, leftTickCount);   // ← N봉 전 일봉 객체
    var rightTick = stock.ticks.Tick(date, rightTickCount);
    if (leftTick == null || rightTick == null) return false;

    var lv = leftTick.GetTickValue(leftType);                // ← 문자열 "20MA"를 실제 숫자로
    var rv = rightTick.GetTickValue(rightType);
    return App.Common.Util.Compare(lv * leftRate, rv * rightRate, op);
}

5.4 비교 연산자

Common/Util.cs:790-814

op 의미 코드
gt left < right (좌가 작다) return left < right;
gte, 이상 left <= right
eq left == right
lt left > right
lte, 이하 left >= right

주의: 변수명이 left, right인데 실제로는 비교 방향이 반전돼 있다. UI 라벨 기준이라 그렇게 만든 것. XML의 Operator=gt ⇒ "왼쪽이 오른쪽보다 작다" 로 동작.

18. 결론

평가 본체는 다음 두 축으로 작동한다:

  1. Key 문자열 디스패치 — XML의 <Key>...</Key> 가 곧 어셈블리 내 IExpression 구현체 한 클래스로 매핑되고, 그 클래스의 Condition(stock, date, ...) 메서드 호출이 평가 단위.
  2. 인터페이스 기반 분기 — ExpressionLoopCore가 구현 인터페이스(ICondition / IConditionGroup / IConditionGroupOr / IGroupRangeIndex / IGroupLine)를 보고 호출 시그니처와 자식 처리 방식을 결정. 그룹식은 자식의 평가 시점(일자)을 결정하고, 사이드 채널(stock.GroupDate1, stock.Lines, stock.Except1 등)을 통해 다른 식과 컨텍스트를 공유.

따라서 새로운 조건식을 추가하려면: 1. IExpression + (ICondition 등 적절한 평가 인터페이스)를 구현하는 UserControl을 만들고, 2. Key 속성에 고유 문자열을 부여하고, 3. ExpressionGroups.ExpressionSort 에 등록하면

리플렉션이 자동으로 잡아주고 XML 직렬화도 그대로 동작한다. 평가 엔진(ExpressionLoopCore) 자체를 수정할 필요가 없는 개방-폐쇄 원칙으로 설계되어 있다.
