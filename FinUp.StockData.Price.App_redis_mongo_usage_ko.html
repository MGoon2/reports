핵심 결론

stockdatapubsub 클러스터는 Kiwoom 실시간 PRICE 데이터를 Redis pub/sub로 발행하는 주 경로에서 직접 사용된다.

  • stockdatapubsub: RedisStockDataHandler가 RedisDatabase.GetSubcriber("stockdatapubsub")로 선택한다.
  • stockdata: Kiwoom 실시간 PRICE publish 주 경로에서는 직접 선택되지 않는다. 공통 Redis DB/cache 접근이나 일부 캐시성 작업에서 기본 클러스터로 쓰일 수 있다.
  • MongoDB: KiwoomRedis가 MongoStockDataHandler 생성에 사용한다. 단 현재 PRICE 경로에서는 Mongo 저장 consumer 시작 코드가 주석 처리되어 실제 insert까지 이어지지 않는다.

Redis 설정 사용 흐름

설정사용 여부사용 방식근거
cluster name="stockdatapubsub" 직접 사용 실시간 가격/호가/거래원 데이터를 Redis pub/sub 채널로 발행하는 subscriber 연결에 사용된다. RedisStockDataHandler.cs:24-27
cluster name="stockdata" 간접/기본 경로 공통 Redis 유틸의 기본 DB/cache 접근 경로로 쓰일 수 있으나, Kiwoom PRICE publish 경로에서는 stockdatapubsub가 명시 사용된다. FinUp.Core.Redis\RedisDatabase.cs, RedisCluster.cs

MongoDB 설정 사용 흐름

설정사용 위치실제 동작근거
MongoDB KiwoomRedis 생성자 ConfigurationManager.AppSettings["MongoDB"]로 읽어 MongoStockDataHandler에 전달한다. KiwoomRedis.cs:65-66, KiwoomRedis.cs:80-85
MongoDBName MongoStockDataHandler DB명에 실행 연도 suffix를 붙인다. 예: StockData2026 MongoStockDataHandler.cs:27-40
컬렉션 MongoStockDataHandler 저장 consumer가 시작되면 T_Price, T_Hoga, T_Trader 컬렉션에 batch insert한다. MongoStockDataHandler.cs:60-73, 104-116

중요: 현재 FIDType=PRICE 경로에서는 _mongoPriceHandler?.StartAsync(...) 호출이 주석 처리되어 있다. 따라서 MongoDB 설정은 읽히고 handler 객체는 만들어질 수 있지만, 실제 저장 consumer는 시작되지 않는다.

근거: KiwoomRedis.cs:126-132, MongoStockDataHandler.cs:86-107

런타임 주의점

항목판단설명
Redis 실제 사용 stockdatapubsub로 압축된 RedisMessage를 PRICE, HOGA, TRADER 채널에 publish하는 구조다. PRICE 경로에서는 PRICE가 핵심이다.
MongoDB 부분 사용 설정은 handler 생성에 쓰이지만 PRICE 저장 루프가 주석이라 insert는 발생하지 않는다.
Mongo queue 리스크 _mongoPriceHandler?.AddPrice(price)는 호출되므로, handler가 생성된 상태에서 consumer가 없으면 내부 BlockingCollection이 drain되지 않을 수 있다.

결론

stockdatapubsub는 Kiwoom 실시간 가격 발행의 직접 Redis 연결 설정이다. stockdata는 이 경로의 직접 publish 대상이 아니라 Redis 공통 DB/cache 계열 설정으로 봐야 한다. MongoDB는 Kiwoom 실시간 저장 handler 생성에 사용되지만, 현재 PRICE 경로에서는 저장 시작 코드가 막혀 있어 실제 MongoDB insert는 수행되지 않는 것으로 판단된다.
