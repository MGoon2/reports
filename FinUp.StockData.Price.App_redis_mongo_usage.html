Analysis date: 2026-05-08

Summary

The stockdatapubsub cluster is directly used for Redis pub/sub publishing of Kiwoom real-time PRICE data.

  • stockdatapubsub: selected by RedisStockDataHandler through RedisDatabase.GetSubcriber("stockdatapubsub").
  • stockdata: not selected by the main Kiwoom PRICE publish path; it is a default/common Redis DB/cache cluster.
  • MongoDB: used to construct MongoStockDataHandler, but the current PRICE path comments out the Mongo save consumer start call.

Important: in the current FIDType=PRICE path, _mongoPriceHandler?.StartAsync(...) is commented out. The MongoDB setting can be read and the handler can be created, but the actual Mongo save consumer is not started.

Conclusion

stockdatapubsub is the direct Redis connection setting for Kiwoom real-time price publication. stockdata should be treated as a common Redis DB/cache cluster rather than the direct publish target for this path. MongoDB is used to construct the Kiwoom real-time save handler, but the current PRICE code path does not start Mongo inserts.
