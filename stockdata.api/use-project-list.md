## 6. 확인/한계

- 본 리포트는 정적 분석 기준이다. 런타임에서 JS 라이브러리/외부 DLL이 동적으로 붙이는 URL, 운영 프록시/IIS rewrite, 배포되지 않은 소스 경로는 실제 호출과 다를 수 있다.
- `api/tv`, `api/v2`, `api/v3` datafeed처럼 base URL만 설정하고 client가 `/history`, `/marks`, `/config` 등을 붙이는 구조는 family endpoint 사용으로 추정했다.
- endpoint 미사용 후보는 “저장소 및 배포 config 정적 검색상 미검출”이라는 의미이며, 외부 시스템/브라우저 북마크/운영 스케줄러/수동 호출 여부는 별도 로그 분석이 필요하다.
- DB 권한 객체명 `StockDataApi`처럼 URL/endpoint 호출과 무관한 SQL 문자열은 프로젝트 URL 사용 근거로 보지 않았다.
- URL literal 없이 설정 모델/key만 보이는 항목은 직접 호출 확정이 아니라 설정 전파 후보로 보아 근거 파일과 함께 유지했다.
- 죽은 endpoint 후보 목록은 본 MD와 동일 경로/동일 파일명의 HTML 리포트에도 같은 표로 반영했다.
