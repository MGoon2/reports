4. 이상 징후 / 확인 필요 (우선순위)

HIGH · 인증서 만료 임박
ACM financeup.co.kr 2026-07-02 만료 (약 74일 후). 만료 전 재발급 필수. 이미 만료된 인증서 4개(*.finup.co.kr 2025-03-28, *.ajk.co.kr 2025-05-06, server 2023-08-03, *.hrinvestment.co.kr 2023-08-27)도 정리.
HIGH · MSK 평문 통신
MSK FinUpLog: ClientAuth null, Encryption In Transit PLAINTEXT. VPC 안 모든 EC2 + VPN 접속자가 인증 없이 Kafka 브로커에 접근 가능 (SG-Kafka1 ingress 10.0.0.0/16 + VPN). TLS + IAM/SASL 도입 검토 필요.
HIGH · RDS 가용성
maindb(prod 메인), predb가 모두 Single-AZ. 장애 시 RPO/RTO 리스크 큼. db-from-maindb-at-202603221548 인스턴스는 Backup 보존 0일.
MED · 관찰성 갭
애플리케이션 로그(Spring Boot / React / 레거시 CS) 가 CloudWatch Logs에 수집되지 않음. 로그 확인은 SSM 접속 후 docker logs / 파일 tail. 신규 서비스 37대에 StatusCheck/CPU 알람 없음.
MED · 네트워크 잔재
  • AjkVPCMain 메인 RT가 이미 삭제된 NAT(nat-0ad0349adc7f35f44)를 참조 — 프라이빗 서브넷 인터넷 불가 상태
  • nat-05163b6afa1990e13이 Private Subnet 안에 배치되어 있음 — 동작 점검 필요
  • vpc-03e4ab8b7564072a2 (AWSKafkaTutorialVPC) 실제 사용 안 함 — 제거 검토
MED · S3 설정
newbiz-sabangnet 버킷 PAB 비활성 + 버킷 정책 없음 — 공개 위험 점검 필요. 모든 버킷 버저닝 비활성.
MED · IAM
장기 미접속 사용자 7명(admindbteam(2023-11), smlee, cloudwatch, Gabia_Read_Only, S3FullAccessUser 등). 액세스 키 활성 여부 점검.
LOW · 알람 잔재
이미 삭제된 RDS(devdb, devfinupedudb) 알람이 INSUFFICIENT_DATA 로 남아 있음. himoonil-EC2-StatusCheckFailed-Count-1~5 5개 알람 정체 미확인.

5. 장애 대응 체크리스트

5.1 서비스 접속 장애 (신규)

  1. ALB alb-internal-finup/alb-internal-api Target Group UnHealthyHostCount 메트릭 — 어느 서비스/인스턴스가 빠져나갔는지 특정
  2. 해당 EC2 StatusCheck, CPU, NetworkIn/Out
  3. EC2에 SSM 접속 → docker ps / docker logs <container> — 애플리케이션 로그 확인 (CloudWatch에 없음)
  4. Route53 내부 존 (internal.finup.co.kr) 레코드가 올바른 ALB를 가리키는지 확인

5.2 서비스 접속 장애 (레거시)

  1. GABIA WAF EC2(FinUp-EC2-WAF1/2-N) 상태 — 외부 유입 필터링 지점
  2. ALB ELBMainExternal/ELBMainInternal/ELBMarsExternal 및 각 TG
  3. 레거시 EC2(WEB1/2, API1/2 등) 개별 확인

5.3 DB 장애

  1. RDS → CloudWatch CPUUtilization, DatabaseConnections, FreeStorageSpace, ReadLatency/WriteLatency
  2. RDS 에러 로그: /aws/rds/instance/maindb/error (25MB), /aws/rds/instance/predb/error (241MB)
  3. Single-AZ이므로 즉시 복구 불가 → 긴급 시 snapshot 복원 (예: 기존 db-from-maindb-at-... 방식)
  4. Stockdata-PG(EC2)는 self-host — i-0abe50b233e46cb0b SSM 접속 후 postgres 프로세스/pg_log 확인

5.4 캐시 장애

  • Redis (레거시): AWS/ElastiCache → CurrConnections, Engine CPUUtilization, Evictions
  • Valkey (신규): 동일 메트릭. RG 단위로 Primary endpoint 조회

5.5 Kafka 장애

  • MSK 알람 KafkaDataLogsDiskUsed > 78% (Target Tracking이 자동 확장 시도)
  • 브로커별 BytesIn/BytesOut, ActiveControllerCount, OfflinePartitionsCount
  • AWS API로 토픽 조회 불가 → Kafka CLI 또는 내부 관리 도구 필요

5.6 배포 장애

  • Jenkins master(i-08e1ff51ad68dacc9) / agent 상태
  • ECR 레포 finup/<service> 에 최신 태그가 올라왔는지 (dev-<sha>-<build>)
  • 대상 EC2 SSM 접속 → docker pull 에러, 디스크 공간(df -h), docker system df

6. 접속/조작 방법

6.1 EC2 접속 (SSM 권장, SSH 키 없이)

aws ssm start-session --target <INSTANCE_ID> --region ap-northeast-2

레거시 Windows 인스턴스는 RDP (GABIA WAF 경유) 또는 AWS Fleet Manager 콘솔 사용.

6.2 내부 도구 접속 (VPN 필수)

  • Jenkins: https://jenkins.internal.finup.co.kr
  • Nexus: https://nexus.internal.finup.co.kr
  • Git: https://git.internal.finup.co.kr
  • dev 서비스: <service>.internal.finup.co.kr (예: api-stock.internal.finup.co.kr)

6.3 로그 보기

CloudWatch Logs에는 애플리케이션 로그가 없음. EC2 직접 접속:

aws ssm start-session --target <INSTANCE_ID> --region ap-northeast-2
# 접속 후
sudo docker ps
sudo docker logs -f --tail 200 <container>
sudo tail -f /var/log/<service>/*.log

6.4 RDS 접속

VPN 접속 후 SQL Server 클라이언트 (SSMS/Azure Data Studio):

  • Server: maindb.cddfbcymycsb.ap-northeast-2.rds.amazonaws.com,1412 (포트 주의: 콤마 뒤 1412)
  • 자격증명: 별도 관리 (Jenkins credentials 또는 팀 내 공유 확인 필요)
