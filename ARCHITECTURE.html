FinUp.StockData.RoboAnalysis.App – 코드 분석 문서

.NET Framework 4.7.2 기반 WinForms 애플리케이션. 사용자가 정의한 XML 조건식(Expression) 을 일별/실시간 주식 시세에 평가하여 조건에 부합하는 종목을 추출하고, AI 요약(Gemini) · 차트 캡처 · 채팅 알림 · DB 기록까지 수행하는 "로보 어드바이저" 시스템.

StockExpressions.cs "검색식(StockExpression)" 잡. RoboAdvisor와 동일하게 DynamicExpression 사용해 종목 추출 후, CefSharp로 차트 캡처(SaveCaptureScreenshotAsync + WrapperImpersonationContext) → SetStockExpressionResultInsert로 저장. 추가로 종목 ↔ 테마 키워드(Radar API) 매핑까지 수행.

3.3 종목 평가 (ExpressionLoopCore)

종목 1개 × 날짜 1개에 대해 루트 식 트리를 평가하는 알고리즘 (DynamicExpression.cs:453-499):

foreach exp in 루트식들 (단, ISuccess 제외):
    if !exp.IsGroup:
        isTrue = ((ICondition)exp).Condition(stock, date)            # 단일봉 평가
    elif exp is IConditionGroupOr:
        isTrue = 자식 중 하나라도 ICondition.Condition true            # OR
    elif exp is IGroupRangeIndex:
        isTrue = GetConditionGroupRangeIndex(stock, date, exp)        # 범위 인덱스 그룹
    elif exp is IGroupLine:
        isTrue = GetConditionGroupLine(stock, date, exp)              # 라인 그룹
    else:                                                              # 기간 범위 그룹
        range = ((IGetRange)exp).GetRange(stock, date)                # ① 범위 산출
        ChildExpression의 ISetRange 자식들에 SetRange(...)             # ② 자식에 범위 주입
        foreach tick in range:
            rangeFowardMoving = range.SkipUntil(tick.date)
            isTrue = ((IConditionGroup)exp).ConditionGroupRange(
                       stock, rangeFowardMoving, date, range.Count)   # ③ 슬라이딩 윈도우
            if isTrue: 자식 ICondition도 동일 tick.date로 검증 + AND
            if 모두 true: break
    if !isTrue: break
return isTrue

핵심은 그룹식이 부모–자식 식 트리의 시간축(범위)을 결정한다는 점. 자식들은 부모가 좁힌 범위 내에서만 동작한다.

3.4 결과 컬럼 동적 생성

ISuccess(예: SUC_DEF) 식이 존재하면 RangeCount × RangeTicks 만큼 1구간_20봉, 1구간_20봉_성공 컬럼을 자동 추가하여 백테스트 그리드에 시점별 수익률을 출력 (DynamicExpression.cs:279-298).

5. 데이터/메모리 흐름 핵심 포인트

  1. 사전 적재: Common.Util.DataLoadAsync가 ① 종목 마스터, ② 일봉(Tick)·분봉(TickMinute), ③ 지수(RoboIndex), ④ 영업일을 모두 메모리(Constants.StockDay/StockIndex/RoboIndex/MarketOpenDateAll)에 로드하고 MA/EMA/StoFastK 등을 미리 계산해 둔다. 식 평가는 캐시된 객체 그래프에서만 동작 → 평가 성능 확보.
  2. 식 인스턴스 = WinForms UserControl: UI 표시 + 평가 로직이 하나의 클래스에 결합. JOB 모드도 using (var ui = new DynamicExpression()) 패턴으로 폼을 띄우지 않은 채 인스턴스만 활용.
  3. 그룹식의 자식 범위 전파: IGetRange → ISetRange로 범위(Tick 리스트)가 부모→자식으로 흘러간다. 동시에 ExpressionGroups.currentExpression이 정적 컨텍스트로 살아 있어 자식 식이 형제·부모 정보를 참조 가능.
  4. 등급 우선순위: RoboAdvisor.SearchJob은 등급 낮은 식부터 평가 → 상위 등급에서 같은 종목이 잡히면 상위 등급으로 승격, 최근 10영업일 중복 종목은 제외.
  5. 알림 채널: ChatAPI(ChatAPIAddress)를 통해 외부 메신저로 발송. 발송 여부는 IsManualSendMessage 플래그.
  6. 차트 캡처: CefSharp(off-screen) + WrapperImpersonationContext로 네트워크 공유 폴더에 PNG 저장.
  7. 로깅: system.diagnostics Trace + Kafka(KafkaUrl) + DB의 OperationTermHistory Alive(60초). 운영 모니터링 용.
