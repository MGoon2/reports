1. 개요

오전 Hikari 커넥션 풀 고갈로 로그인 장애가 발생했으나 선제 감지가 불가능했음. 이를 계기로 앱 내부 메트릭(HikariCP/JVM/HTTP) + 호스트 + AWS 인프라를 통합 관측하는 자체 호스팅 모니터링 스택을 구축.

목표

  • 풀 고갈·JVM Heap 포화·응답 지연을 사전 감지 (트렌드 기반 알람)
  • Spring Boot 전 서비스에 Prometheus 엔드포인트 노출하여 일관된 메트릭 통합
  • Telegram 자동 알람 (기존 봇 재활용)
  • 외부 SaaS 의존 제거, AWS 내부망 완결

결과 요약

  • Prometheus 타겟 48개 모두 up
  • 관측 중인 Spring Boot 서비스 11개: api-stock, api-admin, api-mentor, api-radar, api-finance, api-chat, stockdata-admin, stockdata-core, stockdata-realtime, collector-extra-worker, collector-news-worker
  • 호스트 메트릭 14개 EC2: finup-api1/2, front1/2, biz-api1/2, biz-front1/2, api-radar1/2, stockdata-admin/core/realtime1, api-collectdata1
  • AWS 인프라 메트릭 (CloudWatch Exporter): ALB, RDS, ElastiCache 의 24개 메트릭
  • 알람 룰 17개 (prod 환경 필터링 적용)
  • Grafana 대시보드 7개: FinUp Overview, AWS Infra, JVM Micrometer, Spring Boot APM(2), Node Exporter Full, Cadvisor

7. 운영 런북

일상적인 점검

  1. Grafana → FinUp Overview 대시보드에서 HikariCP Pending / JVM Heap / HTTP 5xx 쪽 스파이크 확인
  2. 알람 노이즈 발생 시 Alertmanager silence 추가 (일시적) 또는 aws-alerts.yml 규칙 수정 (영구적)
  3. 주기적: Prometheus 디스크 사용량 확인 (30일 TSDB, 현재 30GB EBS)

모니터링 EC2 관리

  • 인스턴스 ID: i-0db311ea3d6d4a4af / 10.0.202.65
  • Docker Compose: /opt/monitoring/
  • SSM 접근: aws --profile finup ssm start-session --target i-0db311ea3d6d4a4af
  • 컨테이너 재시작: docker compose restart <service>

자주 있는 장애 대응

Hikari 풀 고갈 알람 수신 시

  1. 어느 서비스인지 확인 — 알람 메시지의 application + host 라벨
  2. Grafana FinUp Overview 에서 hikaricp_connections_pending 트렌드 확인 (급상승 / 점진적 상승 구분)
  3. 긴급 임시 조치: 해당 EC2 SSH → docker restart <api-service> (풀 초기화, 다운타임 ~20초)
  4. 근본 원인 분석: docker logs <api> | grep "leak detection" — 커넥션 누수 위치 확인 (leak-detection-threshold 60s 가 이미 적용됨)

모니터링 EC2 가 다운됐을 때

  • 앱 서비스에는 영향 없음 (exporter 는 사이드카, 앱 독립)
  • 알람만 일시적으로 발송 중단 — 복구 우선순위 낮음
  • 복구: docker compose up -d 로 전체 재기동

CloudWatch Exporter 에러

  • IAM 권한 문제 (EC2-Service-Role 에 CloudWatchReadOnlyAccess 필요)
  • config.yml 문법 에러 — aws_statistics 는 [SampleCount, Average, Sum, Minimum, Maximum] 만 허용. p95 등은 aws_extended_statistics 에 별도 지정
