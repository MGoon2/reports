2. UIType 9 가지 모드 — 같은 EXE, 다른 인격

Program.cs 의 Main() 은 App.config 의 <add key="UIType" value="..."> 값으로 단 하나의 Form 을 띄운다. 즉 한 빌드에서 9 가지 운영 인격이 가능.

UIType진입 Form분류역할핵심 Job
JOB Scheduler 일일 배치 장 종료 후 로보 검색식 실행 → 추천 종목 산출 → 카카오 채팅 발송 → 수익률 추적 RoboAdvisor + RecommandResult + StockExpressions
SIM SchedulerTest 시뮬레이션 사용자 입력 날짜 범위로 과거 일자별 로보 시뮬 (백테스트) RoboAdvisor + RecommandResult
EXP SchedulerExpression 차트 캡처 배치 검색식 결과 종목의 차트를 CefSharp 로 PNG 캡처, 테마 매핑 후 SP 저장 StockExpressions
UI DynamicExpression 디자이너 검색식을 UserControl 트리로 시각 편집하는 메인 디자이너 UI (IExpression 동적 로드)
MR RealtimeScheduler 실시간 모의투자 실시간 — 분봉 갱신 (1 분 주기) → 식 평가 → 신호 publish MockTrade
MS RealtimeSchedulerMars 실시간 US 종목 실시간 1 분봉 검색식 (5 분 주기) + 차트 캡처 Mars
MST RealtimeSchedulerMarsTicker 실시간 US Ticker 일봉 기반 실시간 검색식 (30 초 주기) MarsTicker
UR RealtimeDebuger 디버그 실시간 UI 디버그 — 검색식 + 분봉 시각 검증 MockTrade (수동)
PRO ProfitTest 분석 도구 특정 검색식의 수익률 백테스트 폼 (직접 로직)
주의 — App.config 의 UIType=SIM 은 “현재 운영 모드” 를 뜻하지 않는다.
레포에 체크인된 App.config 는 마지막 개발자 설정일 뿐이다. 운영 머신은 동일 EXE 를 다른 App.config 로 띄울 수 있고, 실제로 JOB(일일 배치) 또는 MR/MS/MST(실시간) 가 별도 머신에서 동시에 떠 있을 가능성이 높다. 운영 호스트별로 어떤 모드인지 확인이 필요하다(§8).

  1. Step 0 — 호스트 식별: §8.3 의 Redis CLIENT LIST 시도. 사용자 승인만 받으면 즉시 결판. 대안: 운영팀 직접 문의가 더 빠름.

    • 5 개 ConnectionString 중 “운영 vs STG” 어느 라인이 활성인지 (체크인본 주석 다수)
  3. Step 2 — 실행 모드 확정: JOB / MR / MS / MST 매트릭스 채워야 “이 머신은 분봉 생성을 한다 / 안 한다” 명확. 분봉 생성 호스트 종료가 RoboAnalysis 폐기 Phase 5 의 트리거.

    • VS 2019+ / .NET Framework 4.7 Developer Pack / WinForms 워크로드
    • x64 빌드 (CefSharp 가 x64 전용)
    • 10.0.x 사설망 도달성 (VPN 또는 SSM Port Forwarding) — Redis/Mongo/MSSQL 모두 사설

    • App.config 의 ConnectionString 5 개 + Redis 3 개 + MongoDB 2 개 + KafkaUrl(MSK) + UNC 자격 — 운영본 회수 필수
  5. Step 4 — 안전한 테스트 시작: UIType=SIM 으로 먼저 띄우면 운영 데이터 쓰기 최소화. 단 IsManualSendMessage=N, IsManual=N 동시 확인. 그 다음 운영 모드로 dry-run.
